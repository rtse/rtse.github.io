What is ES6?

ES6, also known as ECMAScript 2015, is the 6th version of the ECMAScript Language Specification standard.

What is ES6?

ES6 was finalized in June 2015 and adds many new features on top of ES5: classes, modules, arrow functions, promises, for/of loops, etc.

Transpilers

  • A "source-to-source" compiler
  • A transpiler takes a piece of source code and translates to another piece of source code
  • In Javascript's world, ES6 is translated to ES5 through JS transpilers. (Why ES6 to ES5?)
  • Why transpilers? Because browsers do not support all new features (i.e ES6 Module System and Loaders)

Constants + Variables

Variable (var)


function printHello() {
  if(true) {
    var result = "Hello World";
  }
  console.log(result);
}
printHello(); // "Hello World"

var-declared variables are function-scoped.

Variable (let)


function printHello() {
  if(true) {
    let result = "Hello World";
  }
  console.log(result);
}
printHello(); // Exception: ReferenceError: result is not defined

let-declared variables are block-scoped.

Variable (var)


function printNumber() {
  var x = 1;
  if(x === 1) {
   var x = 2;
  }
  console.log(x);
}
printNumber();

Output: 2

Variable (let)


function printNumber() {
  let x = 1;
  if(x === 1) {
   let x = 2;
  }
  console.log(x);
}
printNumber();

Output: 1

Variable (let)

  • let-declared variables are block-scoped.
  • Global let variables are not the properties in the window object (e.g. window.letVariableName).
  • Re-declaring a let variable will result the ReferenceError, instead of undefined.

const


const PI = 3.14159;
const TOTAL = 10000;
TOTAL = 0; // Exception: TypeError
TOTAL++; // Exception: TypeError

const is like let, except that it cannot be re-assigned or you will get the TypeError.

Destructuring

Destructuring arrays


var someNumbers = [1, 2, 3];
var num1 = someNumbers[0];
var num2 = someNumbers[1];
var num3 = someNumbers[2];
console.log(num1 + ", " + num2 + ", " + num3);

Output: 1, 2, 3

Destructuring arrays


var someNumbers = [1, 2, 3];
var [num1, num2, num3] = someNumbers;
console.log(num1 + ", " + num2 + ", " + num3);

Output: 1, 2, 3

Destructuring arrays


var someNumbers = [1, 2, 3];
var [, , numOnly] = someNumbers;
console.log(numOnly);

Output: 3

Destructuring arrays


var [head, ...tail] = [1, 2, 3, 4];
console.log(tail);

Output: [2, 3, 4]
"...tail" refers to the "rest" pattern.

Destructuring arrays


var x = 1;
var y = 2;

[x, y] = [y, x];
console.log(x);
console.log(y);

Output: x => 2, y => 1

Swapping variables becomes brief but comprehensive.
No more temp variables!

Destructuring objects


var { firstName, lastName } = {
	firstName: "Richard",
	lastName: "Tse"
};

console.log(firstName); // Richard
console.log(lastName); // Tse

Destructuring objects


var firstName, lastName;

({ firstName, lastName } = {
	firstName: "Richard",
	lastName: "Tse"
});

console.log(firstName); // Richard
console.log(lastName); // Tse

Note: without the declaration, "(...)" is required to destructure the variables or you will get the SyntaxError.

Modules

  • "A module is simply a JavaScript file written with module syntax. Modules export values, which can then be imported by other modules."
  • ES6 modules are automatically strict-mode code. (No "use strict";)
  • AMD and CommonJs JavaScript files are modules, and they present to ES6 as default export (one per module). For example, when require(...) or define(...) is used.

Modules

export / import

Named export


/------ mathLib.js ------
export function add(x, y) {
    return x + y;
}
export function substract(x, y) {
  return x - y;
}

//------ main.js ------
import {add as sum, substract as minus} as myModule from 'mathLib';
sum(1, 2);
minus(2, 1);

Modules

export / import

Default export (one per module / one per script)


/------ moduleA.js ------
export default function () { ... };

//------ main.js ------
import moduleA from 'moduleA';
//  or import { default as moduleA } from 'moduleA';
moduleA();

Note: if the named export and default export both exist in the same module, then the default export is considered to be the "main" exported value.
